# 페이징 기능 설계 및 유효성 검사 전략

- 날짜: 2026-03-23
- 참여: 사용자 + Gemini CLI

## 배경

게시판 목록의 페이징 기능을 구현함에 있어, 사용자 친화적인 유효성 검사 방식과 서버 측의 0-based/1-based 인덱스 변환 설계를 확립하기 위해 논의함.

## 핵심 논의

- **페이징 인덱스 설계**: 프로그래밍 언어와 DB(LIMIT/OFFSET)의 특성에 따라 내부적으로는 0부터 시작하는 인덱스를 사용하되, 사용자에게는 1부터 보여주는 관례를 따르기로 함.
- **유효성 검사 위치**: 컨트롤러에서 사용자의 요청 페이지 번호를 검증하고, 범위를 벗어난 요청(음수 또는 최대 페이지 초과)을 차단하는 역할을 담당함.
- **응답 전략**: 잘못된 페이지 요청 시 사용자에게 명확한 피드백을 주기 위해 자바스크립트 `alert`와 `history.back()`을 결합한 응답 방식을 채택함. 이는 입력 데이터 보존과 확실한 인지 효과가 있음.
- **공통 유틸리티 활용**: `Resp.java`에 `script()` 메서드를 추가하여 컨트롤러 로직을 깔끔하게 유지하고 재사용성을 높이기로 함.

## 결론

- 컨트롤러에서 1-based 요청을 수신하여 유효성 검사 후 0-based로 변환하여 서비스에 전달.
- 유효성 검사 실패 시 `Resp.script("메시지")`를 통해 즉각적인 팝업 및 이전 페이지 이동 처리.
- UI(Mustache)에서는 `disabled` 속성을 활용해 비정상적인 클릭을 원천 차단.
