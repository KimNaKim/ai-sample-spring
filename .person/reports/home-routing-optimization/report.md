# 홈 페이지 라우팅 및 데이터 바인딩 최적화 보고서

## 1. 개요
브라우저 루트 경로(`/`) 접속 시 404 에러 대신 게시글 목록이 포함된 홈 화면(`home.mustache`)을 정상적으로 렌더링하도록 서버 로직을 개선함.

## 2. 주요 변경 사항

### 2.1 라우팅 설정 변경 (`BoardController.java`)
- 기존 `/home` 매핑을 제거하고 루트 경로(`/`) 및 `/board` 경로를 추가 매핑함.
- `Model` 객체를 통해 `home.mustache`가 기대하는 `models` 데이터를 바인딩함.

### 2.2 비즈니스 로직 추가 (`BoardService.java`)
- `BoardRepository`를 통해 모든 게시글을 최신순(`id` DESC)으로 조회하는 `findAll()` 메서드 구현.
- 서비스 레이어에서 엔티티를 DTO(`BoardResponse.Max`)로 변환하여 반환하도록 설계(프로젝트 규칙 준수).

### 2.3 DTO 확장 (`BoardResponse.java`)
- 목록 및 상세 조회를 위한 표준 DTO인 `Max` 클래스를 추가함.
- 엔티티를 인자로 받는 생성자를 통해 데이터 변환 로직을 캡슐화함.

## 3. 결과 및 기대 효과
- 사용자가 메인 페이지 접속 시 게시글 존재 여부와 상관없이 항상 일관된 UI(홈 화면)를 경험할 수 있음.
- 게시글이 없는 경우에도 `home.mustache` 내의 조건부 렌더링(`{{^models}}`)을 통해 친절한 안내 문구를 제공함.
